# Catacomb Startup Status

## Verdict

Catacomb is the strongest umbrella repo for the Overandor capital-formation thesis. It is not merely a repo scanner. It is a deterministic intervention engine for discovering underpriced software assets, scoring revival potential, and converting neglected repositories into monetizable infrastructure.

## Category

Catacomb belongs in a new category:

**Software Asset Capitalization Engine**

It searches for hidden software value, produces evidence-backed intervention scores, and generates deterministic transformation paths.

## Core Thesis

Most open-source and private software assets are mispriced because their value is not measured continuously. Catacomb turns repositories into scored intervention targets by combining factual evidence, buildability checks, novelty detection, market-demand signals, and strategy generation.

The economic primitive is:

```text
repository
→ evidence scan
→ buildability score
→ novelty score
→ market-demand score
→ revival score
→ intervention path
→ transformation plan
→ capitalizable software asset
```

## Why Catacomb Is Stronger Than WaveMint / Capitalizer

WaveMint sounds like a token minter.
Capitalizer sounds generic.
Catacomb sounds like discovering buried value.

That matches the actual product:

**Catacomb finds buried software assets and revives them into capital.**

## Startup Wedge

Initial wedge:

**Find abandoned, underexposed, high-utility repositories that can be revived, packaged, and monetized.**

Target customers:

1. Solo builders looking for acquisition/revival targets.
2. Micro-SaaS operators searching for productizable open-source infrastructure.
3. Venture studios mining technical assets.
4. AI agents looking for high-expected-value coding interventions.
5. Funds or service firms doing software due diligence.

## Product Surface

### CLI

A deterministic command-line scanner for topics, users, and specific repos.

### API

A hosted endpoint that returns revival scores, intervention paths, evidence hashes, and opportunity reports.

### Dashboard

A visual ranked board of hidden infrastructure assets.

### LiquidDB Export

Every scan can be written into a portable `.db` file with records, evidence, scores, hashes, and intervention histories.

### Snapshot Layer

Each scan batch can be compressed, hashed, uploaded to IPFS, and committed as a verifiable software-opportunity wave.

## Proof-of-Wave Fit

Catacomb is the data engine that can feed Proof-of-Wave.

A Catacomb scan creates a wave:

```text
scan batch
→ scored repos
→ evidence ledger
→ intervention plans
→ LiquidDB snapshot
→ IPFS CID
→ wave manifest
→ contributor credits
```

The worker is whoever triggers, validates, enriches, or acts on the scan.

## Current Strengths

- Clear deterministic-agent positioning.
- Existing CLI entrypoint.

- Multi-agent architecture.
- Direct link to monetizable repo intervention.

## Current Gaps

- Needs durable SQLite/LiquidDB output.
- Needs snapshot/export mode.
- Needs API server.
- Needs CI and tests.
- Needs package metadata.
- Needs hosted dashboard.
- Needs real evidence schemas.
- Needs contribution ledger if connected to tokenized rewards.

## MVP Definition

Catacomb reaches MVP when it can:

1. Scan a GitHub topic, user, or repo.
2. Store every result in SQLite.
3. Produce deterministic evidence hashes.
4. Rank repos by intervention value.
5. Export a JSON report.
6. Export a compressed `.db` snapshot.
7. Serve `/repo/{owner}/{repo}` and `/scan/{id}` pages.
8. Produce a machine-readable manifest for each scan.

## Startup-Grade Claim

Catacomb should not claim to predict guaranteed returns.

The safe claim is:

**Catacomb ranks software-revival opportunities using deterministic evidence and explicit intervention assumptions.**
